My personal setup

General setup

Most of my personal computing is done on a Thinkpad X230 from 2012.

  • A quite fresh openSUSE Tumbleweed as operating system/distro
  • GNOME as desktop environment, no customizations
  • dotfiles managed with git and GNU Stow
  • Syncthing to Peer-to-Peer sync some files across different devices
  • Passwords managed with KeePassXC (synced with Syncthing)
  • Yubikey device and yubikey-oath for 2FA on Desktop and mobile
  • Firefox as Web browser, with some extensions
    • Multi-Account Containers to isolate websites
    • Facebook Container to isolate this particular website
    • KeePassXC extension (to integrate with password manager)
    • uBlock Origin (Ad blocker)
  • NextCloud to host shared files
  • NextCloud clients to sync files with NextCloud on desktop and mobile
  • Org-mode (“Your life in text mode”) on top of Emacs to organize notes, journals and information. It also powers this website.
    • Synced with Syncthing
    • Orgzly as TODO widget on Android
    • Orgro as Org file viewer on Android
  • tarsnap for backups

You can also read some details about the setup for my music hobby activities.

I also have a small Home “Lab”

  • 2 Raspberry Pi 3 running openSUSE Leap 15.1
  • 1 Pine64 running OpenBSD 6.7
  • HomeAssistant
  • Managed by salt-ssh

Work

Most of my work happens on a Thinkpad X220 from 2011, except for video-conferencing, for which I use my iPad, as the laptop is not powerful enough.

Work mail setup

  • mbsync to synchronize IMAP locally
  • msmtp to send email
  • mu (maildir-utils) as mail indexer/search
  • mu4e on top of Emacs as mail front-end/composer
  • plugins/utilities
    • outlook package to reply HTML email

You can find this setup in my internal dotfiles repository.

Other

  • unofficial onedrive client to sync files to OneDrive
  • restic for backups
  • Bose 700 noise cancelling headphones for a great confcall experience
  • Jabra Speaker for mobile conferences involving more than one person in the room

Development environment

  • Emacs as editor for most languages, with many extensions. Some highlights:
    • LSP mode to take advantage of all LSP servers out there
    • Magit as a git front-end
  • tmux to manage my consoles and to split the screen in my favourite layout (editor left side, two command lines right side)
  • Given my open-source background, I have to dig in code writen in many languages. My go-to languages are C++, Golang and Ruby depending on the task.

C++

  • CMake as build system and to fetch dependencies
  • clang provides most of the features to lsp-mode in Emacs

Ruby

  • I use the distribution interpreter and bundler to vendor dependencies inside the project
  • I use the Solargraph gem to power lsp-mode in emacs

Golang

  • I just use the standard Go modules
  • gopls to power lsp-mode in emacs
